Arrays and Hashes

January 1, 2016

Arrays and hashes are two of the most powerful and commonly used objects in Ruby, particularly when building apps. This is because one of the most common functions of both arrays and hashes, which is called iteration, allows you to call a block of code on an infinitely large series of inputs, and making our lives much easier.

There are several ways to create an array, but the easiest one for beginners is the literal array constructor, which is done using the brackets [].

In this example, you'll see that we can create an array, arr, just by setting it equal to brackets []. Arrays can be filled with anything, even including other arrays. While this is helpful, the true power is in iteration, which we'll get to later. For now, just understand that arrays are essentially numbered lists of things.

Hashes are very similar to arrays, but differ in a very key way: instead of being a list single items with an assigned number (well, you could nest arrays within arrays, but the whole new array is treated as a single object), hashes actually store data in key: value pairs. You always must have both a key and a value - although the value can be nil, it must always be explicitly stated. Now to get to one of the most powerful functions of both arrays and hashes: iteration. Iteration is the languages ability to apply a block of code to every element in the object. The syntax for each is a little different based on how they're structured, and how they behave is a little different, but the concept is the same for both: for each element the object has, do what you tell it to do.

As you can see, the code for the hash is a bit more complicated because you're dealing with both the key and the value, and you can do different things with each, while the array is just one object per element to focus on. In both cases, however, we were able to write just a single block that will now repeat itself over as many items as we want to put into our containers. There are many more, powerful methods that can be used on arrays and hashes, but that is a challenge for another day.
